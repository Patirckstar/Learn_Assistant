# AI 学习助手 - 开发文档

---

## 一、项目概述

基于 LangChain 的智能学习辅助系统，支持知识库构建、课程大纲生成、逐章学习、在线测验、错题复盘、人脸识别登录、语音交互等功能。采用前后端分离架构。

---

## 二、技术架构

| 向量数据库 | ChromaDB | 0.5+ |
| 关系数据库 | MySQL 9.5 | — |
| 数据库驱动 | PyMySQL | 1.1+ |

### 4.2 表结构设计

```sql
-- ========================
-- 1. 用户表 (users)
-- ========================
CREATE TABLE users (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    password_hash   VARCHAR(255),                 -- 备用密码登录
    face_encoding   TEXT,                         -- 人脸特征编码（npy文件路径）
    face_image_path VARCHAR(500),                 -- 注册时的人脸照片路径
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 2. 文档表 (documents)
-- ========================
CREATE TABLE documents (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    filename    VARCHAR(255) NOT NULL,
    file_type   VARCHAR(20) NOT NULL,             -- pdf / docx / txt / md / ppt / json
    file_path   VARCHAR(500) NOT NULL,            -- 文件存储路径
    file_size   INT,                              -- 文件大小(字节)
    chunk_count INT DEFAULT 0,                    -- 分块数量
    user_id     INT,
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 3. 课程章节表 (chapters)
-- ========================
CREATE TABLE chapters (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    parent_id   INT,                              -- 父章节(章)，NULL=顶级章
    title       VARCHAR(255) NOT NULL,
    level       INT NOT NULL DEFAULT 1,           -- 1=章, 2=节
    sort_order  INT DEFAULT 0,
    content     TEXT,                             -- LLM生成的章节学习内容
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES chapters(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 4. 学习进度表 (progress)
-- ========================
CREATE TABLE progress (
    id           INT AUTO_INCREMENT PRIMARY KEY,
    user_id      INT NOT NULL,
    chapter_id   INT NOT NULL,
    status       VARCHAR(20) DEFAULT 'not_started', -- not_started / learning / completed
    started_at   DATETIME,
    completed_at DATETIME,
    UNIQUE KEY uk_user_chapter (user_id, chapter_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 5. 题目表 (questions)
-- ========================
CREATE TABLE questions (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    chapter_id  INT,
    type        VARCHAR(20) NOT NULL,              -- single_choice / true_false
    difficulty  VARCHAR(10) DEFAULT 'medium',      -- easy / medium / hard
    stem        TEXT NOT NULL,                     -- 题干
    options     JSON NOT NULL,                     -- 选项（MySQL JSON类型）
    answer      VARCHAR(255) NOT NULL,             -- 正确答案
    explanation TEXT,                              -- 解析
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 6. 考试记录表 (exams)
-- ========================
CREATE TABLE exams (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    user_id     INT NOT NULL,
    chapter_id  INT,
    total_score DECIMAL(5,1) NOT NULL,
    score       DECIMAL(5,1) NOT NULL,
    time_limit  INT,                              -- 时间限制(秒)
    time_used   INT,                              -- 实际用时(秒)
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 7. 答题明细表 (exam_details)
-- ========================
CREATE TABLE exam_details (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    exam_id     INT NOT NULL,
    question_id INT NOT NULL,
    user_answer VARCHAR(255),
    is_correct  TINYINT(1),
    score       DECIMAL(4,1),
    FOREIGN KEY (exam_id) REFERENCES exams(id) ON DELETE CASCADE,
    FOREIGN KEY (question_id) REFERENCES questions(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ========================
-- 8. 错题本表 (wrong_book)
-- ========================
CREATE TABLE wrong_book (
    id                INT AUTO_INCREMENT PRIMARY KEY,
    user_id           INT NOT NULL,
    question_id       INT NOT NULL,
    chapter_id        INT,
    exam_id           INT,
    wrong_count       INT DEFAULT 1,              -- 错误次数
    correct_count     INT DEFAULT 0,              -- 后续答对次数
    last_wrong_at     DATETIME,
    last_practiced_at DATETIME,
    UNIQUE KEY uk_user_question (user_id, question_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (question_id) REFERENCES questions(id) ON DELETE CASCADE,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id) ON DELETE SET NULL,
    FOREIGN KEY (exam_id) REFERENCES exams(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 六、核心业务流程

### 6.1 知识库构建流程

```
用户上传文档 → 后端接收并保存文件 → 文件类型判断
  → PDF: PyPDFLoader 解析文本
  → DOCX: python-docx 提取文本
  → MD / TXT: 直接读取文本内容
  → PPT: python-pptx 提取文本
  → JSON: json.load 解析结构化内容
  → 文本分块 (RecursiveCharacterTextSplitter, chunk_size=500, overlap=50)
  → sentence-transformers 向量化
  → 存入 ChromaDB collection
  → 返回成功，记录文档信息到 MySQL
```

### 6.2 出题流程

```
用户选择章节 + 难度 + 数量
  → 从 ChromaDB 检索该章节相关知识点
  → 拼接 Prompt 调用 LLM
  → LLM 返回结构化题目（JSON）
  → 解析并存入 questions 表
  → 返回题目列表给前端
```

### 6.3 考试批改流程

```
用户答题 → 提交答案
  → 逐一比对用户答案与正确答案
  → 生成考试记录 (exams)
  → 生成答题明细 (exam_details)
  → 错题自动归档到 wrong_book
  → 返回得分、正确答案、解析
```

### 6.4 人脸登录流程

```
用户打开摄像头 → 拍照/录制
  → （可选）活体检测：眨眼检测/摇头检测
  → 图片上传后端 → face_recognition 提取特征编码
  → 与数据库中已注册的人脸特征逐一比对（欧氏距离）
  → 距离 < 阈值(0.6) → 认证通过，返回 token
  → 距离 >= 阈值 → 拒绝，提示重试或使用密码登录
```

## 九、约定规范

### 9.1 代码风格
- 后端：PEP 8，使用 Black 格式化
- 前端：ESLint + Prettier，Vue 3 Composition API
- 命名：变量/函数 camelCase，类 PascalCase，数据库字段 snake_case

### 9.2 Git 分支策略
- `main` — 稳定发布分支
- `develop` — 开发主分支
- `feature/模块名` — 功能分支（如 `feature/knowledge-base`）
- Commit 格式：`type: 描述`，如 `feat: 完成文档上传解析功能`

### 9.3 配置文件
- 所有环境相关配置放 `.env` 文件，不提交到 Git
- 提供 `.env.example` 模板
