# AI 学习助手 - 开发文档

---

## 一、项目概述

基于 LangChain + Ollama（Qwen2.5）的智能学习辅助系统，支持知识库管理、在线测验、学习进度追踪、人脸识别登录、语音指令控制等功能。前后端分离架构。

## 四、数据库设计

### 4.1 实体关系

```
users ──── user_profiles
  │
  ├─── documents         (用户上传的文档)
  ├─── chapter_progress  (学习进度)
  ├─── exams ─── exam_details ─── questions
  ├─── wrong_book ────── questions
  └─── papers ────────── chapters
```

### 4.2 表结构

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| `users` | 用户 | id, username, password_hash, face_encoding, face_image_path |
| `user_profiles` | 用户资料 | id, user_id, nickname, avatar, phone |
| `chapters` | 章节(树形) | id, parent_id, title, content, level, sort_order |
| `documents` | 文档 | id, user_id, filename, file_type, chunk_count |
| `questions` | 题目 | id, chapter_id, type, difficulty, stem, options(JSON), answer |
| `papers` | 试卷 | id, chapter_id, title, question_count, is_ready |
| `exams` | 考试记录 | id, user_id, chapter_id, total_score, score, time_used |
| `exam_details` | 答题明细 | id, exam_id, question_id, user_answer, is_correct |
| `wrong_book` | 错题本 | id, user_id, question_id, wrong_count, correct_count |
| `chapter_progress` | 学习进度 | id, user_id, chapter_id, status, time_spent |

### 4.3 关键约束

- `wrong_book` 每用户每题一条记录（`UNIQUE(user_id, question_id)`）
- `chapters` 支持父-子树形结构（`parent_id` 自引用外键）
- `exams` 通过 `chapter_id` 关联试卷，不再直接依赖 `papers` 表
- 所有表使用 `InnoDB` 引擎 + `utf8mb4` 字符集

## 五、API 接口设计

失败时返回 `{"detail": "错误信息"}`（HTTP 状态码 4xx/5xx）。

## 六、核心业务流程

### 6.1 知识库构建流程

```
用户上传文档 → FastAPI 接收 → 保存到 data/uploads/
  → file_parser 解析文本（PDF/DOCX/PPTX/TXT/MD）
  → task_queue 异步任务：文本分块 → 向量化 → 存入 ChromaDB
  → 更新 documents 表的 chunk_count
  → 返回上传成功
```

### 6.2 出题流程（试卷刷新）

```
用户点击"刷新试卷"
  → 后端遍历所有父章节（level=1）
    ├─ 已有 paper 记录 → 跳过
    └─ 无 paper → 调用 Ollama LLM 生成题目
       → 解析 JSON 返回（单选 + 判断）
       → 存入 questions 表
       → 创建 paper 记录
  → SSE 流式进度推送给前端
```

### 6.3 考试批改流程

```
用户答题 → submit_exam_for_paper()
  → 逐题比对用户答案与正确答案
  → 计算每题分值 (round(100/N, 1))
  → 创建 exams 记录（动态 total_score）
  → 创建 exam_details × N 条答题明细
  → 错题自动写入 wrong_book（合并计数）
  → LLM 生成 AI 学习建议
  → 返回结果给前端
```

### 6.4 人脸识别流程

```
录入：摄像头拍照 → base64 → 后端 face_recognition 提取 128 维特征编码
  → 存入 users.face_encoding（JSON 数组字符串）

登录：摄像头拍照 → base64 → 后端提取特征 → 遍历比对（欧氏距离）
  → 距离 < 0.6（FACE_TOLERANCE）→ 登录成功，返回 JWT
  → 距离 ≥ 0.6 → 拒绝登录
```

### 6.5 语音指令流程

```
用户按住麦克风 → Web Speech API 录音 → 松开 → 获取文字
  → 用户在弹窗中可编辑文字
  → 发送 POST /api/voice/command {text}
  → LLM 解析意图（导航/闲聊）
    ├─ 导航 → 返回 action=navigate + target URL → 前端跳转
    └─ 闲聊 → 返回 action=chat + reply → 前端显示回复
```

## 七、开发环境搭建

### 7.1 环境准备

```bash
# Ollama
ollama pull qwen2.5:7b
ollama serve

# MySQL
CREATE DATABASE learn_assistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 7.2 后端启动

```bash
cd backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cp .env.example .env         # 编辑 .env 中的 DB_PASSWORD
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### 7.3 前端启动

```bash
cd frontend
npm install
npm run dev
```

### 7.4 Embedding 模型

项目使用 `sentence-transformers/all-MiniLM-L6-v2` 模型进行文本向量化。

本地路径：`backend/models/all-MiniLM-L6-v2/`

可通过两种方式获取：

```bash
# 方式一：从 HuggingFace 自动下载（首次运行时）
cd backend
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2', cache_folder='./models')"

# 方式二：手动下载并解压到 backend/models/all-MiniLM-L6-v2/
```

## 八、安全设计

| 方面 | 措施 |
|------|------|
| 密码存储 | bcrypt 哈希（passlib） |
| 接口认证 | JWT（python-jose），Bearer Token |
| 用户隔离 | API 通过 JWT 提取当前用户，数据查询均带 user_id 过滤 |
| 人脸特征 | 存储在数据库（非文件系统），避免被直接盗取 |
| CORS | 限制仅前端 dev server 域名可跨域访问 |
| 文件上传 | 限制 50MB，不暴露服务器路径 |
| `.env` | 已加入 `.gitignore`，禁止提交到仓库 |

## 九、约定规范

| 规范 | 说明 |
|------|------|
| Python 版本 | 3.11+ |
| 命名约定 | 变量/函数 `snake_case`，类 `PascalCase`，URL `kebab-case` |
